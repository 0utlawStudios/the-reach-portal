Truth From The Workflow

The supplied V2.7 workflow contains 14 nodes. It runs every 5 minutes, reads posts from Supabase, filters due posts in CST, then publishes Facebook, Instagram, and LinkedIn.

  • Get Scheduled Posts: Supabase getAll on posts.
  • Filter: stage=eq.approved_scheduled, scheduled_date and scheduled_time present.
  • Publish Facebook: Graph API v21, images/videos/reels.
  • Publish Instagram: Graph API v21, image/video/reel/carousel.
  • Publish LinkedIn: legacy /v2/ugcPosts, image only.

Safety Notes

Must patch before live

V2.7 marks a post as posted before platform publishing starts. The current Reach app is stricter: a post should move to Posted only after platform API success with posted_at and posted URLs.

V2.7 also logs to legacy post_audit_logs. The current app uses audit_log_v2. This handoff gathers access only; the workflow wiring should be updated later.

Exact SMM Checklist

Use this as the message to Stefani and the SMM. Check items as access is collected. The checklist is saved after refresh.

  1. Ask Stefani for Meta admin access.
    Ask: "Please add Aldridge/the implementation email to The Reach Meta Business Portfolio with access to the Facebook Page and the linked Instagram Business account. We need enough access to generate or authorize a Page token for publishing."
  2. Go to Meta Business settings.
    Open business.facebook.com/settings. Under People, verify the implementation email is added. Under Business assets, verify access to The Reach Facebook Page and The Reach Instagram account.
  3. Collect Facebook Page details.
    Get the official Page URL and Page ID. Put the Page ID in FB_PAGE below. This is the exact value used by the V2.7 Facebook node.
  4. Collect Instagram Business Account ID.
    Confirm the Instagram profile is Business/Creator and linked to the Page. Get the Instagram Business Account ID and put it in IG_USER below.
  5. Create or authorize the Meta Page token.
    Use the Meta app/publishing setup to generate a token that can publish to the Page and linked Instagram account. Put that token in FB_TOKEN and IG_TOKEN, or enter the vault item name if secrets are stored elsewhere.
  6. Ask Stefani for LinkedIn Page admin access.
    Ask: "Please add Aldridge/the implementation email as an admin on The Reach LinkedIn Company Page so we can authorize organization posting."
  7. Collect LinkedIn Organization ID.
    Open the LinkedIn Company Page admin area. Get the Organization ID and format it as urn:li:organization:YOUR_ORG_ID. Put it in LI_ORG below.
  8. Create or authorize the LinkedIn access token.
    Use the LinkedIn app/OAuth setup with organization posting permission. Put the token in LI_TOKEN, or enter the vault item name.
  9. Prepare public media test files.
    V2.7 reads source_vault.rawFiles and converts Google Drive share links to direct download links. Test media must be reachable by platform APIs. Confirm "Anyone with the link" access for test files.
  10. Save this form.
    Fill the boxes below, press Save + Download TXT, and send the file named THE_REACH_N8N_AUTOPUBLISH_HANDOFF.txt.
