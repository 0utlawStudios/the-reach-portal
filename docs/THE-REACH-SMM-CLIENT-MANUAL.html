01 - Operating rhythm

From asset to published post.

The app is built around one simple rule: no post should silently vanish, skip approval, or publish without the required creative context. The safest workflow is upload, create, review, approve, publish, archive.

1

Upload Media

Add files to Media Library or upload during Create Post. Files are backed by Google Drive.

2

Create Post

Pick platform, content type, caption, date/time, asset source, and source vault details.

3

Review

Use the drawer comments, notes, and checklist to prepare the post for approval.

4

Approve

Only complete posts can move to Approved / Scheduled. Revision requests require notes.

5

Publish

The publisher claims due jobs, posts to connected platforms, records attempts, and archives URLs.

Daily

Check Dashboard, review posts awaiting approval, and clear support tickets or mentions.

Weekly

Review the calendar, fill date gaps, upload fresh media, and confirm every scheduled post has a caption and source.

Before client demo

Open Dashboard, Pipeline, Post Preview, Media Library, Brand Kit, and Support. Confirm cards move and data saves after refresh.

03 - Content Pipeline

Where the work actually moves.

The pipeline uses draggable cards with server-backed stage changes. A move is not just visual: it writes to Supabase, updates other screens through Realtime, and records audit history.

StageWhat belongs thereMove rule
IdeasDraft concepts, new uploads, posts not ready for formal review.Can move forward when the post has enough content to review.
Awaiting ApprovalPosts ready for the approver to inspect.Approver can approve or request revision.
Revision NeededPosts blocked by creative, copy, source, or timing feedback.Returning to Awaiting Approval requires a revision note.
Approved / ScheduledComplete posts approved for the scheduled date and time.Requires media, caption, design link, schedule, asset source, and checklist completion.
PostedPublished content with recorded publish metadata.Manual drag into Posted is blocked; publisher service marks posted.

Realtime behavior: when one approved user drags a card, other active screens receive the change through Supabase Realtime. If the database rejects a move, the UI rolls the card back instead of pretending it worked.

04 - Create Post

Every required field has a purpose.

The Create Post modal is intentionally strict. It prevents posts from reaching approval with missing media rights, missing schedule data, or missing source context.

Fastest clean workflow

  • Click Create Post.
  • Choose Browse Library if the asset is already uploaded.
  • Select platforms and content type.
  • Write the caption and use @mentions if a teammate needs attention.
  • Set date/time and asset source.
  • Add design link, Drive folder, license/release if needed, and notes.
  • Complete the checklist before sending to approval.

Required to demo drag approval

A card can move into Approved / Scheduled only when the production fields are filled.

Platform first, format second.

The app disables invalid platform/content-type pairings so the team does not build a post that cannot be published cleanly. If a platform fades out, switch the content type or remove incompatible platforms.

06 - Media Library + Drive

The asset vault behind the board.

Media Library is the cleanest way to prepare for post creation. Upload once, then reuse through Browse Library. The production backend is connected to the Google Drive asset store, with source files and license references kept on the post.

60 TB backend

Drive is the durable storage layer for publishing media and large source files.

Small + large uploads

Small files proxy through the server; large files use resumable Drive upload sessions.

Source vault

Design links, raw files, usage type, and Drive folder stay attached to each card.

Rights proof

License, release, or screenshot proof can be uploaded for compliance.

SMM trick: upload batches by campaign before creating posts. Then use Create Post -> Browse Library to pick the exact asset. This saves repeated upload time, keeps Drive organized, and makes demo cards draggable without "missing media source" errors.

07 - Post Preview

Preview prevents avoidable mistakes.

Post Preview shows platform-style renderings before the post goes live. It is useful because the same caption and media can feel different on Instagram, Facebook, TikTok, YouTube, and LinkedIn.

Visual fit

Quickly check whether the image or video thumbnail reads well at platform scale.

Caption rhythm

See whether the first line, hashtags, and CTA feel natural before publishing.

Approver confidence

Approvers can review the post in context, not just as a file and a text box.

10 - Brand Kit + Settings

The shared operating standard.

Brand Kit stores the copy patterns and visual rules that make the portal useful for repeatable social work. Settings controls team access, themes, account integrations, audit logs, and publishing operations.

Brand Kit includes

  • Business essentials and website reference.
  • Hashtag banks and caption templates.
  • Hooks, CTAs, content pillars, and voice rules.
  • Reach colors: Sand, Stone, Sun, Water.
  • Logo assets, social specs, guardrails, and approval chain.

Settings includes

  • Default light theme, with dark mode saved per user preference.
  • Team members, invites, resend invites, roles, avatars, and last-seen state.
  • Audit logs for important post and system activity.
  • Publishing queue monitoring and admin retry for failed/stuck jobs.

12 - Admin and production controls

What keeps the portal hardened.

The app uses Supabase Auth, workspace membership, RLS, server-side support routes, and protected health checks so access and data integrity are enforced beyond the browser UI.

Invites and resend

Superadmins invite users, resend pending invites, and activate team profiles through the setup flow.

Forgot password

Known Auth users receive reset links. Known team members without Auth can receive a setup bridge without exposing unknown emails.

Last seen

Presence, route pings, heartbeat, departure beacons, and audit sync contribute to team status labels.
